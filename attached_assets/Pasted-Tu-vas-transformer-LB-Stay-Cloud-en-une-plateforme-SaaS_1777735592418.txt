## ARCHITECTURE MULTI-TENANT
Crée un système de rôles :
- SUPER_ADMIN : voit et gère TOUTES les enseignes de toutes les industries
- OWNER : propriétaire d'une ou plusieurs enseignes
- MANAGER : gère une branche spécifique
- STAFF : caissier/employé avec accès limité

Ajoute dans Supabase une table "businesses" avec : id, name, sector, owner_id, plan, logo_url, color_theme, created_at, is_active
Ajoute une table "branches" liée à businesses.
Le SUPER_ADMIN voit un dashboard global avec toutes les enseignes.
